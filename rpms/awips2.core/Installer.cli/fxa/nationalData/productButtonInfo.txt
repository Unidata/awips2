#  productButtonInfo.txt

#  This table describes product buttons. A product button, when 
#  pressed, loads a depict key that matches the current scale.
#
#  Each product button key must be unique. List it in the first column.
#
#  Each product button maps to a depict key indexed by scale. It is
#  assumed by default that the user is providing a list of depict keys
#  for each scale, starting with scale 0. If all that is provided is
#  plain depict keys, the last key provided will be repeated if fewer
#  keys are provided than there are scales.  Exception: A tag "^1" means
#  to keep adding keys until all scales are filled, incrementing by one
#  for each scale. Similarly, "v1" means the same, but decrement.
#
#  The user may change this default interpretation of key lists by
#  providing fields that begin with a less than (<) or greater than (>)
#  symbol. Numbers that follow these symbols are assumed to be distances
#  in km if they contain a decimal point or are larger than twice the
#  number of scales; otherwise they are assumed to be scale indices. A
#  scale index of -1 means the last scale, -2 the second to last, etc.
#  A < precedes the minimum distance or the first scale index to
#  associate with a depict key. A > precedes the maximum distance or the
#  last scale index to use.
#
#  A key that immediately follows one or more distances is assigned to
#  all scales having sizes between the maximum and minimum distances.
#  These default to 1e37 and 0, respectively, so it is meaningful to
#  just provide one distance. This leaves the scale index set to the
#  next scale after the last one assigned in this manner. A key that
#  immediately follows one or more scale indices will be assigned to
#  that range of scale indices. The last scale index defaults to 0 and
#  the first scale index defaults to whichever scale would normally be
#  assigned next. A key will always at a minimum be assigned to the first
#  scale index.
#
#  Mixing scale indices and distances between depict keys is not
#  recommended as the result is not well defined.
#
#  For example, if there are 5 scales, and a product button has depict
#  keys (101, 101, 101), then we assume that the fourth and fifth
#  scales are also 101.  If a product button has depict keys (101,
#  105, 109), then we assume the fourth and fifth are 0 and 0.
#
#  Also, if a product button should be disabled at a certain scale,
#  list a zero for the depict key.
#
#  List the button label in the third column. List the toggle menu
#  label (which currently isn't used) in the fourth column. (Labels for
#  plots and history list are found in depictInfo.)
#
#  Finally, put a `y', `Y', or a `1' in the fifth column if this is a
#  static entry.  A static entry is one that appears with no green
#  time in the menus.  Nonstatic entries have green times that get
#  registered for auto-updates with the notification server.
#
